# Rules for this project 
1. No decision is default or assumed, user has to approve anything that isn't already explicitly defined. 
2. No shorthand variables like 'i' or 'e', use 'index', 'example', etc. Code should be human readable. No inline comments, every functional block of code should have a short comment documentation preceeding it.
3. Don't touch unrealted code even if it can be improved, unless it is necessary for the task. 
4. No feature or change can be tier 0, iterate and refine every change at least once before even showing it to the user and aim to achive the simplest solutions first.   
5. Aim to reuse code methods as much as posible, if anything feels bespoke to a specific problem we need to ask ourselves a question - can this be generalised? 
6. Flag uncertainity explicitly and suggest better ways to do things. We don't want to do things with overconfidence but we are open to good ideas you are confident about.
